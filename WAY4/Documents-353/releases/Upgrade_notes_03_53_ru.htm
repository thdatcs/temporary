CARDS-70085

В аппликациях c ActionType =  Add/AddOrUpdate/Update и ObjectType = Client изменилась обработка поля ShortName. Если в апликации оно не заполнено, то в клиенте оно будет сформировано из полей LastName, FirstName, MiddleName разделенных пробелом. Если через аппликации меняется одно из трех полей: LastName, FirstName, MiddleName, то поле ShortName будет пересчитано.
Для изменения разделителя в поле ShortName необходимо задать параметр CLIENT_SHORT_NAME_SEPARATOR
Upgrade Note:
Для отключения нового поведения нужно выставить N глобальному параметру APPL_CALC_CLIENT_SHORT_NAME.

CARDS-61838

Добавлена проверка параметра клиента или контракта, для которого будет выполняться действие "Disable". Если такой параметр не существует, будет отображаться сообщение об ошибке.
Upgrade Note:
Для заявлений c action_type ="Disable", object_type = "ContractParameter"/"ClientParameter" добавлена проверка наличия данного параметра у объекта.  Таким образом, поведение системы изменилось: ранее проверка не выполнялась, поэтому заявления успешно обрабатывались. Новое поведение: в случае если параметр не найден, возникает ошибка с кодом "2300" и заявление отклоняется. 
Предыдущее поведение не поддерживается.
Updated Manuals: [application_management\batch_interface\XML_Applications_R2.pdf]

CARDS-63098

В модуле Application R2 изменена обработка заявлений на создание эквайринговых  контрактов и контрактов устройств. Поля SIC и MerchantID/Card Acceptor ID не наследуются от контракта вышестоящего уровня.
Upgrade Note:
Изменена обработка заявлений на создание эквайринговых  контрактов и контрактов устройств. Предыдущее поведение: если в заявлении на создание контракта не заполнены поля SIC и MerchantID/Card Acceptor ID, то они наследовались с контракта вышестоящего уровня. Таким образом, эти поля были заполнены для каждого контракта в иерархии. Новое поведение: теперь наследование данных полей не происходит. Они могут быть заполнены на любом уровне иерархии контрактов (например, только на самом верхнем).
Updated Manuals: [application_management\r2\acquiring\Advanced_Applications_R2_Acquiring.pdf]

CARDS-64467

В модуле Advanced Applications R2 добавлена возможность переопределить глобальный параметр "APPL_EMPTY_DPRT" на уровне финансового института.
В заявлении на изменение финансового института клиента или контракта параметр "APPL_EMPTY_DPRT" должен иметь одинаковые значения для нового и предыдущего институтов.
Upgrade Note:
До обновления Cards  на версию 03.50.00 необходимо:
  1. Определить и разметить финансовые институты, для которых будет переопределен параметр "APPL_EMPTY_DPRT". 
  2. В случае если хотя бы для одного финансового института используется режим обязательного указания Branch, необходимо выявить контракты и клиентов, у которых не заполнен параметр Branch. Для выявления клиентов и контрактов необходимо запустить скрипты CARDS-65098_Check_Empty_Client_Branch.sql и CARDS-65098_Check_Empty_Contract_Branch.sql соответственно. Скрипты регистрируют в журнале информацию о клиентах и контрактах с пустым полем "Branch". 
Режим обязательного указания поля Branch для финансовго института используется, когда соблюдено хотя бы одно из условий:
  - глобальный параметр "APPL_EMPTY_DPRT" отсутствует или имеет значение "N" и нет переопределений на уровне финансового института;
  - параметр "APPL_EMPTY_DPRT" имеет значение "N" на уровне финансового института.
  3. Для указанных в журнале клиентов и контрактов необходимо вручную заполнить поле "Branch".
Check Scripts:
Чтобы определить наличие некорректной конфигурации, выполните скрипты CARDS-65098_Check_Empty_Client_Branch.sql , CARDS-65098_Check_Empty_Contract_Branch.sql

CARDS-67277

Поддержано заявление на создание контракта с параметром <Currency> с валютой, отличающейся от валюты Продукта.
Upgrade Note:
Раньше отправка элемента <Currency> в заявлениях была возможна без специальной настройки Продукта. Это приводило к созданию контракта в валюте в соответствии с заявлением, однако счета таких контрактов создавались в валюте, заданной по умолчанию на Схеме Счетов. Теперь такие заявления будут обрабатываться только при условии специальной настройки на уровне Продукта, и в этом случае счета контракта будут создаваться в нужной валюте.
Старое поведение не поддерживается, заявления в валюте, отличающейся от валюты, заданной по умолчанию на Схеме Счетов в настройке Продукта будут отклоняться, если на уровне Продукта не выполнена специальная настройка.
Updated Manuals: [application_management\r2\acquiring\Advanced_Applications_R2_Acquiring.pdf] [application_management\batch_interface\XML_Applications_R2.pdf]
