CARDS-67277

Supported an application for contract creation with <Currency> parameter specifying a currency that is different from the Product currency.
Upgrade Note:
Earlier it was possible to send <Currency> element in an application without special Product setup. This resulted in contract creation with the currency as per application, however, contract accounts were created in the default currency as per Account Scheme setup. Now these applications will only be processed when special setup is made on the Product level and contract accounts will be created in the appropriate currency.
Old behavior is not supported, applications with the currency other than the Account Scheme's default currency in Product setup will be declined if special setup is not made in a Product.

CARDS-67979 / OWS-29997

Fixed an issue when the TARIFF_FROM_TRANS_DATE tag in a tariff or INST_FROM_TRANS_DATE tag in a service was considered as FROM_TRANS_DATE tag in a service which led to incorrect interest calculation after posting.
Upgrade Note:
Previously, if the INST_FROM_TRANS_DATE or TARIFF_FROM_TRANS_DATE tag was applied for a transaction, but no FROM_TRANS_DATE tag was specified, the system calculated interest from the transaction date instead of the posting date. Now in the aforementioned configuration, interest is calculated from the posting date. To support the previous behavior it is recommended to specify the FROM_TRANS_DATE tag with the corresponding value. For more information, see the standard manual \core\Setup_Tags.pdf.
Check script could be used to find affected configuration on services and tariffs.
